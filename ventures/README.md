# ventures/

Each venture gets its own folder here: `ventures/<slug>/` containing:
- `STATE.md` — the venture's memory (see
  .claude/skills/forge-playbook/references/state.md for the format)
- `architecture.md` — the architect's design
- `product.md` — pm's backlog (T2+)
- `deliverables/` — marketing + sales assets (positioning.md, pricing.md, etc.)
- `compliance.md` — risk register (T4)

Created automatically by /assess after you approve the tier. Commit these to
keep your venture history; they compound across sessions.
