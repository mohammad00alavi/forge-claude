# Business deliverable quality rubric

Business outputs need a gate too, or the marketer/sales agents grade their own
homework. A separate reviewer pass (the strategist or a dedicated reviewer)
scores each deliverable. This is the business-track equivalent of typecheck +
test for code.

## Universal checks (every deliverable)

- [ ] **Specific, not generic.** Could this copy belong to any product in the
      category? If yes, it's filler — reject. Name the actual user, the actual
      pain, the actual outcome.
- [ ] **Claims are grounded.** No invented stats, no fabricated testimonials,
      no "trusted by thousands" unless true. Mark assumptions as assumptions.
- [ ] **One clear action.** Each asset has a single primary CTA / next step.
- [ ] **Consistent with the product.** Features named match what was actually
      built (pull from pm's final feature list, not the original pitch).

## Positioning / messaging doc
- [ ] One-sentence value prop a stranger could repeat.
- [ ] Target segment is narrow enough to be real ("indie iOS devs" not
      "developers").
- [ ] Differentiation states what this does that alternatives don't.
- [ ] Objections anticipated and answered.

## Landing copy
- [ ] Headline states the outcome, not the mechanism.
- [ ] Above-the-fold answers: what is it, who's it for, why care.
- [ ] Proof/credibility section (honest — roadmap if pre-launch).
- [ ] Pricing is legible; the plan differences are obvious.

## GTM / launch plan
- [ ] Channels match where the target user actually is.
- [ ] Sequenced (pre-launch → launch → post) with concrete actions.
- [ ] Each action is something the human can execute or approve, not vague
      ("post on relevant communities" → names the communities).

## Sales assets (T3+)
- [ ] Pricing tied to value metric, not pulled from air.
- [ ] Outreach templates are personalizable, not spam.
- [ ] Objection handling addresses the real top-3 objections.

## Verdict
Each deliverable: **PASS**, **REVISE** (numbered concrete fixes), or
**ESCALATE** (needs a human decision — e.g., a pricing number, a legal claim).
Generic-but-not-wrong still fails: "could be sharper" is not the bar; "names
the actual user and outcome" is.
